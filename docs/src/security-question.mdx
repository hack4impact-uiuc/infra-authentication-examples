# Security Questions



## Security Question For User Endpoint

| Endpoint | /securityQuestionForUser [POST] |
| ------ | ----------- |
| Body   | email |
| Response 200 | security question |
| Response 400 | invalid request |

(1) If getting the security message was unsuccessful, will return a 400 response with message "Invalid request".

(2) If a user with that email doesn't exist in the database, will return a 400 response with message "User is not registered!".

(3) If all is successful, will return the security question.

```json
{
	"email": "abc@def.com"
}
```

Example response:
```json
{
    "question": "What street did you grow up on?"
}
```


## Get Security Questions Endpoint

| Endpoint | /getSecurityQuestions [POST] |
| ------ | ----------- |
| Body   | token |
| Response 200 | security question |
| Response 400 | invalid request |
| Response 500 | server error |

Example body (no headers required for this endpoint):
```json
{
	"token": "eyJhbGciOiJIUzI.eyJ1c2VySWQiOiI1Y2M0YWFhZGVhZDAyCdQNDdlODEwN2IiLCJoYXNoZWRQYXNzd29yZCI6IiQyYiQxMCRvU0xjU3l1Z3hvdGF5NFVOVnNoWjJPU1ZIYlAzaERSwiZXhwIjoxNTU2NDc5MDIxfQ.RQTCZ0-Ldu6iGHvOAOXQWXTrsuVb1kUF4ItHTp3RpKk"
}
```

Example response:
```json
{
    "questions": [
        "What is your mother's maiden name?",
        "What is your favorite color?",
        "What street did you grow up on?",
        "What was the name of your first grade teacher?"
    ]
}
```
(1) If the request body is invalid, it will return with message "invalid request".

(2) If getting the security message was unsuccessful, will return a 500 response with message "No security question could be parsed from config file".

(3) If all is successful, will return an array of questions that are used for security.
